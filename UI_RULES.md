# UI Rules

## Consistency
- Use a 4px/8px spacing rhythm.
- Keep radius and shadow tokens consistent.
- Maintain a clear typography hierarchy.

## States
- Always handle loading, empty, error, success, and disabled states.
- Provide visible focus indicators for keyboard users.

## Components
- Standardize buttons, inputs, cards, tables, and navigation.
- Prefer reusable primitives over one-off styles.

## Accessibility
- Ensure sufficient contrast.
- Use semantic HTML and proper labels.
