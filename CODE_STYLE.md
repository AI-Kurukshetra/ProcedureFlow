# Code Style

## General
- Prefer clarity over cleverness.
- Keep functions small and single-purpose.
- Avoid unnecessary abstractions.

## Next.js
- Preserve server/client boundaries.
- Add `"use client"` only when required.
- Keep data fetching on the server when possible.

## TypeScript
- Prefer explicit types for public APIs.
- Avoid `any` unless there is no alternative.
- Keep types close to usage.

## Styling
- Follow existing styling conventions (Tailwind/CSS Modules/Component library).
- Avoid inline styles when a class can be used.
